Build Personal Website with Hugo

May 10 2022 · 2 min read
#website

This is a brief introduction about how to build a personal website with hugo! The following steps has worked on my Ubuntu20.04.

Table of Contents
  • Install Hugo
  • Create Blog with Hugo
    • Choose a Theme
  • Deploy to Github

Install Hugo

Open a terminal, and run sudo apt-get install hugo. Then, run hugo version to see if the installation is successful. If it’s successful, it should output the version of hugo, mine is

Hugo Static Site Generator v0.68.3/extended linux/amd64 BuildDate: 2020-03-25T06:15:45Z

Create Blog with Hugo

Run hugo new site myblog to create a new site named myblog.

Run cd myblog to enter the blog.

Choose a Theme

Choose a theme from Hugo Theme, and follow its installation guide. (Remember to install and config your git when downloading the theme.)

See your blog locally by running hugo serve. The default path is http://localhost:1313/, you can open it in the browser. Then write the blog and modify other config files to see what has changed. After satisfying with your modification, deploy it to remote server so that others may see it.

Deploy to Github

Create a new repository named by 'your-github-name'.github.io

Open config.toml and modify baseURL = "https://'your-github-name'.github.io" in line 1.

Run hugo in myblog which will generate a public folder. Run the following command:

cd public
git init
git add .
git commit -m "myblog"
git remote add origin https://github.com/'your-github-name'/'your-github-name'.github.io.git
git push -u origin +master

Enter your user-name and password, then I got

remote: Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.

To generate a personal access token, you have to go to github.com, login yout account, click settings -> Developer settings -> Personal access tokens -> Generate new token, then run git push -u origin +master again, and fill in the new token instead of password when asking to fill in passward.

Afterwards, if you make some update to your blogs, run the following commonds:

cd myblog
hugo
cd public
git add .
git status
git commit -m "update blogs"
git push

To simplify, youcan write a shell, for example, touch blosh.sh and write the following into the shell:

cd myblog
hugo
cd public
git add .
git status
git commit -m "update blogs"
git push
